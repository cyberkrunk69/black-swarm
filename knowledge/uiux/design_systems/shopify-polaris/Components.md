# Components

Components are the reusable building blocks for creating Shopify admin experiences.

## Actions

- [![Account connection](/_next/image?url=%2Fimages%2Fcomponents%2Factions%2Faccount-connection.png&w=3840&q=75)

  #### Account connection

  Used for connecting or disconnecting a store to various accounts, like Facebook for the sales channel.](/components/actions/account-connection)
- [![Button](/_next/image?url=%2Fimages%2Fcomponents%2Factions%2Fbutton.png&w=3840&q=75)

  #### Button

  Used primarily for actions like 'Add', 'Close', 'Cancel', or 'Save'. Plain buttons are used for less important actions.](/components/actions/button)
- [![Button group](/_next/image?url=%2Fimages%2Fcomponents%2Factions%2Fbutton-group.png&w=3840&q=75)

  #### Button group

  Displays multiple related actions stacked or in a horizontal row for arrangement and spacing.](/components/actions/button-group)

## Layout and structure

- [![Bleed](/_next/image?url=%2Fimages%2Fcomponents%2Flayout-and-structure%2Fbleed.png&w=3840&q=75)

  #### Bleed

  Applies negative margin to a layout, extending it to the edge of the screen on small screens.](/components/layout-and-structure/bleed)
- [![Block stack](/_next/image?url=%2Fimages%2Fcomponents%2Flayout-and-structure%2Fblock-stack.png&w=3840&q=75)

  #### Block stack

  Use to display children vertically and horizontally with full width by default. Based on CSS Flexbox.](/components/layout-and-structure/block-stack)
- [![Box](/_next/image?url=%2Fimages%2Fcomponents%2Flayout-and-structure%2Fbox.png&w=3840&q=75)

  #### Box

  Box is the most primitive layout component. It’s a way to access Polaris design tokens.](/components/layout-and-structure/box)
- [![Callout card](/_next/image?url=%2Fimages%2Fcomponents%2Flayout-and-structure%2Fcallout-card.png&w=3840&q=75)

  #### Callout card

  Callout cards are used to encourage merchants to take an action related to a new feature or opportunity. They are most commonly displayed in the sales channels section of Shopify.](/components/layout-and-structure/callout-card)
- [![Card](/_next/image?url=%2Fimages%2Fcomponents%2Flayout-and-structure%2Fcard.png&w=3840&q=75)

  #### Card

  Used to group similar concepts and tasks together for easier scanning and reading.](/components/layout-and-structure/card)
- [![Divider](/_next/image?url=%2Fimages%2Fcomponents%2Flayout-and-structure%2Fdivider.png&w=3840&q=75)

  #### Divider

  Use to separate or group content.](/components/layout-and-structure/divider)
- [![Empty state](/_next/image?url=%2Fimages%2Fcomponents%2Flayout-and-structure%2Fempty-state.png&w=3840&q=75)

  #### Empty state

  Empty states are used when a list, table, or chart has no items or data to show. This is an opportunity to provide explanation or guidance to help merchants progress. The empty state component is intended for use when a full page in the admin is empty, and not for individual elements or areas in the interface.](/components/layout-and-structure/empty-state)
- [![Form layout](/_next/image?url=%2Fimages%2Fcomponents%2Flayout-and-structure%2Fform-layout.png&w=3840&q=75)

  #### Form layout

  Manages the layout of all forms and fields within it. Used for the layout of new forms and managing the layout of all forms.](/components/layout-and-structure/form-layout)
- [![Grid](/_next/image?url=%2Fimages%2Fcomponents%2Flayout-and-structure%2Fgrid.png&w=3840&q=75)

  #### Grid

  Create complex layouts based on CSS Grid.](/components/layout-and-structure/grid)
- [![Inline grid](/_next/image?url=%2Fimages%2Fcomponents%2Flayout-and-structure%2Finline-grid.png&w=3840&q=75)

  #### Inline grid

  Use to lay out children horizontally with equal gap between columns. Based on CSS Grid.](/components/layout-and-structure/inline-grid)
- [![Inline stack](/_next/image?url=%2Fimages%2Fcomponents%2Flayout-and-structure%2Finline-stack.png&w=3840&q=75)

  #### Inline stack

  Use to display children horizontally in a row. Based on CSS Flexbox.](/components/layout-and-structure/inline-stack)
- [![Layout](/_next/image?url=%2Fimages%2Fcomponents%2Flayout-and-structure%2Flayout.png&w=3840&q=75)

  #### Layout

  A structural component used to group other components. Creates consistent spacing and helps layout stack and scale responsively.](/components/layout-and-structure/layout)
- [![Media card](/_next/image?url=%2Fimages%2Fcomponents%2Flayout-and-structure%2Fmedia-card.png&w=3840&q=75)

  #### Media card

  Provides a container for introductory or highlight information. Often used in a grid to present related content.](/components/layout-and-structure/media-card)
- [![Page](/_next/image?url=%2Fimages%2Fcomponents%2Flayout-and-structure%2Fpage.png&w=3840&q=75)

  #### Page

  Used to build the layout of a page in the Shopify admin. A flexible container for composing pages consistently.](/components/layout-and-structure/page)

## Selection and input

- [![Autocomplete](/_next/image?url=%2Fimages%2Fcomponents%2Fselection-and-input%2Fautocomplete.png&w=3840&q=75)

  #### Autocomplete

  The autocomplete component is an input field that provides selectable suggestions as a merchant types into it. It allows merchants to quickly search through and select from large collections of options. It's a convenience wrapper around the Combobox and Listbox components with minor UI differences.](/components/selection-and-input/autocomplete)
- [![Checkbox](/_next/image?url=%2Fimages%2Fcomponents%2Fselection-and-input%2Fcheckbox.png&w=3840&q=75)

  #### Checkbox

  Checkboxes are most commonly used to give merchants a way to make a range of selections (zero, one, or multiple). They may also be used as a way to have merchants indicate they agree to specific terms and services.](/components/selection-and-input/checkbox)
- [![Choice list](/_next/image?url=%2Fimages%2Fcomponents%2Fselection-and-input%2Fchoice-list.png&w=3840&q=75)

  #### Choice list

  A choice list lets you create a list of grouped radio buttons or checkboxes. Use this component if you need to group together a related list of interactive choices.](/components/selection-and-input/choice-list)
- [![Color picker](/_next/image?url=%2Fimages%2Fcomponents%2Fselection-and-input%2Fcolor-picker.png&w=3840&q=75)

  #### Color picker

  Allows merchants to choose a color visually, or by entering a hex value.](/components/selection-and-input/color-picker)
- [![Combobox](/_next/image?url=%2Fimages%2Fcomponents%2Fselection-and-input%2Fcombobox.png&w=3840&q=75)

  #### Combobox

  Combobox is an accessible autocomplete input that enables merchants to filter a list of options and select one or more values.](/components/selection-and-input/combobox)
- [![Date picker](/_next/image?url=%2Fimages%2Fcomponents%2Fselection-and-input%2Fdate-picker.png&w=3840&q=75)

  #### Date picker

  Date pickers let merchants choose dates from a visual calendar that’s consistently applied wherever dates need to be selected across Shopify.](/components/selection-and-input/date-picker)
- [![Drop zone](/_next/image?url=%2Fimages%2Fcomponents%2Fselection-and-input%2Fdrop-zone.png&w=3840&q=75)

  #### Drop zone

  The drop zone component lets users upload files by dragging and dropping the files into an area on a page, or activating a button.](/components/selection-and-input/drop-zone)
- [![Filters](/_next/image?url=%2Fimages%2Fcomponents%2Fselection-and-input%2Ffilters.png&w=3840&q=75)

  #### Filters

  A composite component that filters the items of a list or table.](/components/selection-and-input/filters)
- [![Form](/_next/image?url=%2Fimages%2Fcomponents%2Fselection-and-input%2Fform.png&w=3840&q=75)

  #### Form

  A wrapper component that handles the submission of forms.](/components/selection-and-input/form)
- [![Index filters](/_next/image?url=%2Fimages%2Fcomponents%2Fselection-and-input%2Findex-filters.png&w=3840&q=75)

  #### Index filters

  Use index filters to allow merchants to filter, search, and sort their index table data and create unique saved views from the results.](/components/selection-and-input/index-filters)
- [![Inline error](/_next/image?url=%2Fimages%2Fcomponents%2Fselection-and-input%2Finline-error.png&w=3840&q=75)

  #### Inline error

  Inline errors are brief, in-context messages that tell merchants something went wrong with a single or group of inputs in a form. Use inline errors to help merchants understand why a form input may not be valid and how to fix it.](/components/selection-and-input/inline-error)
- [![Radio button](/_next/image?url=%2Fimages%2Fcomponents%2Fselection-and-input%2Fradio-button.png&w=3840&q=75)

  #### Radio button

  Use radio buttons to present each item in a list of options where merchants must make a single selection.](/components/selection-and-input/radio-button)
- [![Range slider](/_next/image?url=%2Fimages%2Fcomponents%2Fselection-and-input%2Frange-slider.png&w=3840&q=75)

  #### Range slider

  A range slider is an input field that merchants can use to select a numeric value within a given range (minimum and maximum values).](/components/selection-and-input/range-slider)
- [![Select](/_next/image?url=%2Fimages%2Fcomponents%2Fselection-and-input%2Fselect.png&w=3840&q=75)

  #### Select

  Select lets merchants choose one option from an options menu. Consider select when you have 4 or more options, to avoid cluttering the interface.](/components/selection-and-input/select)
- [![Tag](/_next/image?url=%2Fimages%2Fcomponents%2Fselection-and-input%2Ftag.png&w=3840&q=75)

  #### Tag

  Tags represent a set of interactive, merchant-supplied keywords that help label, organize, and categorize objects. Tags can be added or removed from an object by merchants.](/components/selection-and-input/tag)
- [![Text field](/_next/image?url=%2Fimages%2Fcomponents%2Fselection-and-input%2Ftext-field.png&w=3840&q=75)

  #### Text field

  A text field is an input field that merchants can type into. It has a range of options and supports several text formats including numbers.](/components/selection-and-input/text-field)

## Tables

- [![Data table](/_next/image?url=%2Fimages%2Fcomponents%2Ftables%2Fdata-table.png&w=3840&q=75)

  #### Data table

  Used to organize and display all information from a data set. Aimed to be as simple as possible for merchants.](/components/tables/data-table)
- [![Index table](/_next/image?url=%2Fimages%2Fcomponents%2Ftables%2Findex-table.png&w=3840&q=75)

  #### Index table

  An index table displays a collection of objects of the same type, like orders or products. The main job of an index table is to help merchants get an at-a-glance of the objects to perform actions or navigate to a full-page representation of it.](/components/tables/index-table)

## Lists

- [![Action list](/_next/image?url=%2Fimages%2Fcomponents%2Flists%2Faction-list.png&w=3840&q=75)

  #### Action list

  Action lists render a list of actions or selectable options. This component is usually placed inside a popover container to create a dropdown menu or to let merchants select from a list of options.](/components/lists/action-list)
- [![Description list](/_next/image?url=%2Fimages%2Fcomponents%2Flists%2Fdescription-list.png&w=3840&q=75)

  #### Description list

  Used to present pairs of related information, like terms and definitions, or names and values, in a list format.](/components/lists/description-list)
- [![List](/_next/image?url=%2Fimages%2Fcomponents%2Flists%2Flist.png&w=3840&q=75)

  #### List

  Lists display a set of related text-only content. Each list item begins with a bullet or a number.](/components/lists/list)
- [![Listbox](/_next/image?url=%2Fimages%2Fcomponents%2Flists%2Flistbox.png&w=3840&q=75)

  #### Listbox

  A Listbox is a vertical list of interactive options, with room for icons, descriptions, and other elements.](/components/lists/listbox)
- [![Option list](/_next/image?url=%2Fimages%2Fcomponents%2Flists%2Foption-list.png&w=3840&q=75)

  #### Option list

  The option list component lets you create a list of grouped items that merchants can pick from. This can include single selection or multiple selection of options. Option list usually appears in a popover, and sometimes in a modal or a sidebar. Option lists are styled differently than choice lists and should not be used within a form, but as a standalone menu.](/components/lists/option-list)
- [![Resource item](/_next/image?url=%2Fimages%2Fcomponents%2Flists%2Fresource-item.png&w=3840&q=75)

  #### Resource item

  Resource items represent specific objects within a collection, such as products or orders. They provide contextual information on the resource type and link to the object’s detail page.](/components/lists/resource-item)
- [![Resource list](/_next/image?url=%2Fimages%2Fcomponents%2Flists%2Fresource-list.png&w=3840&q=75)

  #### Resource list

  A resource list displays a collection of objects of the same type, like products or customers. The main job of a resource list is to help merchants find an object and navigate to a full-page representation of it.](/components/lists/resource-list)

## Navigation

  #### Link

- [![Tabs](/_next/image?url=%2Fimages%2Fcomponents%2Fnavigation%2Ftabs.png&w=3840&q=75)
